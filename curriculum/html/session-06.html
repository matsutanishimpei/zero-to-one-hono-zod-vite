第6回：Cloudflare D1（データベース）

今日の目標: Bindings の仕組みを理解し、D1 に対して CRUD 操作を行う。

1. Bindings とは？

wrangler.json で Worker と DB を紐付けると、c.env.DB でアクセスできるようになります。

{
  "d1_databases": [{
    "binding": "DB",
    "database_name": "my-db",
    "database_id": "xxx"
  }]
}

2. Hono で型を定義

type Bindings = { DB: D1Database }
const app = new Hono<{ Bindings: Bindings }>()

3. スキーマ定義

CREATE TABLE IF NOT EXISTS messages (
  id INTEGER PRIMARY KEY AUTOINCREMENT,
  content TEXT NOT NULL,
  created_at TEXT DEFAULT (datetime('now'))
);

4. CRUD 操作

// 取得
app.get('/messages', async (c) => {
  const { results } = await c.env.DB
    .prepare('SELECT * FROM messages ORDER BY created_at DESC')
    .all()
  return c.json(results)
})

// 作成
app.post('/messages', zValidator('json', z.object({
  content: z.string().min(1).max(140)
})), async (c) => {
  const { content } = c.req.valid('json')
  await c.env.DB
    .prepare('INSERT INTO messages (content) VALUES (?)')
    .bind(content).run()
  return c.json({ status: 'ok' }, 201)
})

まとめ

  • wrangler.json で Binding を設定 → c.env.DB でアクセス。
  • Hono<{ Bindings }> で型を付けると補完が効く。
  • prepare().bind().all() / .run() で SQL を実行。
