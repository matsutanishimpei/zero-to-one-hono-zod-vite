第3回：Zodによるデータバリデーション

今日の目標: Zodでのスキーマ定義と、不整合なデータの自動除去を学ぶ。

1. バリデーションの重要性

「防御的なプログラミング」として、サーバー側での入力チェックは必須です。

2. Zodによるスキーマ定義

import { z } from 'zod'

const userSchema = z.object({
  name: z.string().min(1, '名前は必須').max(20),
  age: z.number().int().positive(),
  email: z.string().email()
})

3. Honoとの連携

import { zValidator } from '@hono/zod-validator'

app.post('/register', zValidator('json', userSchema), (c) => {
  const data = c.req.valid('json')
  return c.json({ message: `${data.name}さん、登録完了！` })
})

ハンズオン

わざとエラーになるデータ（年齢がマイナス、メール形式でない等）を送り、どのようレスポンスが返るか確認しましょう。
