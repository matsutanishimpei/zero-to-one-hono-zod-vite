特別回：モノレポ構成とコード共有

今日の目標: モノレポ形式でプロジェクトを整理し、バリデーションロジックをフロント・バック間で共有する。

1. モノレポ（Monorepo）とは？

複数のパッケージを一つのリポジトリで管理する手法です。共通の「型」や「バリデーションルール」を共有するのに非常に有効です。

2. ディレクトリ構成

my-project/
├── package.json
└── packages/
    ├── backend (Hono)
    ├── frontend (Vite)
    └── shared (Zod schemas)

3. npm workspaces 設定

{
  "name": "my-monorepo",
  "private": true,
  "workspaces": ["packages/*"]
}

4. 共有パッケージの利用

// packages/backend/src/index.ts
import { userSchema } from '@my-app/shared'

// packages/frontend/src/App.tsx
import { userSchema } from '@my-app/shared'

まとめ

  • npm workspaces を設定して管理。
  • 共有パッケージを作ることで、API仕様の変更がフロントに即座に伝わる。
  • ../../ の相対パス地獄から解放される。
